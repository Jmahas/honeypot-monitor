PREPARACION DEL ENTORNO

Guía paso a paso para que tu honeypot funcione
1️⃣ Preparar entorno

Instalar Python 3.11+ (o compatible).

Instalar dependencias:

pip install PyQt5 PyQtWebEngine flask requests

PyQt5 → GUI

PyQtWebEngine → para el mapa en la GUI

Flask → servidor web del dashboard

requests → si tu código hace llamadas HTTP (opcional)

Asegúrate de estar en la carpeta del proyecto (donde están main.py, compartir/, servidor/):

cd "F:\Programacion\DEUSTO\Python\Proyecto_finalPython"
2️⃣ Verificar estructura del proyecto

Debes tener algo así:

Proyecto_finalPython/
│-- compartir/
│   └-- eventos.py
│-- servidor/
│   └-- ssh_server.py
│-- mapa.html
│-- main.py (o gui.py)
│-- README.md

event_queue debe estar definido en compartir/eventos.py como un queue.Queue().

start_server() debe estar en servidor/ssh_server.py.

start_web() debe iniciar Flask:

def start_web():
    web_app.run(host="0.0.0.0", port=5000, debug=False, use_reloader=False)
3️⃣ Iniciar GUI y servidor SSH

Ejecuta tu GUI principal (main.py o como lo llames):

python main.py

La GUI hará automáticamente:

Lanzar el servidor SSH en un hilo separado.

Lanzar el servidor Flask en otro hilo.

Mostrar tabla y mapa.

Mensajes en consola que debes ver:

[*] Interfaz iniciada correctamente.
[*] Honeypot SSH escuchando en puerto 2222
[*] Dashboard web en http://localhost:5000
4️⃣ Probar SSH localmente

Desde otra terminal (o máquina en la misma red):

ssh -p 2222 usuario@127.0.0.1

Cambia usuario por lo que quieras probar (admin, root, etc.)

Pon cualquier contraseña.

Tu GUI debería actualizar la tabla y el mapa con la IP, usuario, contraseña y ciudad/pais simulados.

5️⃣ Probar ataques desde otra máquina

En otra máquina de la misma red:

ssh -p 2222 root@IP_DE_TU_PC

La GUI debe actualizar con la IP de la otra máquina.

Si quieres simular ataques masivos, puedes usar un simple bucle con ssh y diferentes usuarios/contraseñas.

6️⃣ Abrir dashboard web

En tu PC:

http://localhost:5000

Verás el mapa y los ataques que llegan en tiempo real.

Cada ataque de event_queue aparecerá en el mapa con lat/lon y la tabla.

7️⃣ Exponer dashboard con Cloudflared (opcional)

Si quieres ver el mapa desde otra red:

Instala Cloudflared: https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/

Exponer el puerto 5000:

cloudflared tunnel --url http://localhost:5000

Cloudflared te dará una URL tipo:

https://randomstring.trycloudflare.com

Abre esa URL desde cualquier dispositivo y verás el dashboard en tiempo real.

8️⃣ Seguridad y funcionamiento

Solo para pruebas, no lo expongas a Internet sin protección.

Los datos de IP/usuario/contraseña son ficticios o de prueba.

Si quieres, puedes usar threading.Lock() para events_buffer para evitar problemas de concurrencia:

events_lock = threading.Lock()
with events_lock:
    events_buffer.append(e)

Escapa correctamente los datos que pasas a JavaScript:

import json
self.web.page().runJavaScript(
    f"addAttack({lat}, {lon}, {json.dumps(ip)}, {json.dumps(ciudad_info)})"
)
